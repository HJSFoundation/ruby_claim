The claim file produced by ProMedical is formatted as a JSON object array.

Each entry in the main array is a hash (dictionary) representing each claim. The individual claim hash contains all of the information relevant to the entire claim. The claim hash also contains an array of services with each record being another hash of the individual service level data elements (up to but not exceeding 6).

Field type may be only one of the following types:

    string      Will be a string containing the value specified.

    boolean:    Will be either 'true' or 'false'

    money:      Will be represented as a a float.  The dollar sign is not included.

    checkbox:   If field is checkbox its options will be specified.  One of the options defined below will be sent.

    date:       Will be a valid date formatted with four digit year then a dash then two digit month then a dash then a two digit day.  ( ie '2010-01-01' )

    phone:      Will be 10 digits with no separation.


****   Diagnosis codes will also be sent as an array of values.  This is very similar to service data, but will only contain the values.  ( ie ['V103.01','102.2'] )
